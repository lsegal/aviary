# Reference

The reference section is split in two tracks.

## UI Reference

- [Control Panel Surface](./ui/control-panel) maps the major browser-visible areas and what they expose.

## MCP Reference

- [MCP Index](./mcp/) groups the callable tool surface by domain.
- Each MCP page below is a placeholder that should expand into request schemas, examples, return values, permissions, and failure modes.

## Reference Policy

- One page should exist for every meaningful user-visible function area.
- Tool names should stay aligned with the code registry.
- Breaking changes are acceptable, so the docs should track the current implementation directly.
